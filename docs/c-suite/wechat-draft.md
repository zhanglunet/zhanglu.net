# AI 原生组织的 C-suite，需要一套什么样的决策智能体

> 我先后做了 Boss、MBA Brand、OAF 三个看起来不相关的产品，做到第三个才发现它们是同一套骨架。这篇讲讲这套骨架，以及为什么 AI 原生组织的 C-suite 需要它。

## 三个产品，一个意外的发现

就在最近一个月，我先后做了三个 AI 产品，面向的人完全不同：

- **Boss**（bossagent.cc）—— 给 CEO 的战略 OS。抛一个战略议题，它开一场多评委合议，产出一份带归因校验的判断书。
- **MBA Brand**（mbabrand.com）—— 给 CMO 的品牌影响力审计。给一个品牌名，它跑完多维度调研，再让一组评委独立打分。
- **OAF**（oaf.world）—— 给 CFO 的投研工作台。覆盖美 / A / 港三市场，把数字交给确定性工具、把叙事交给大模型。

领域不同、用户不同、数据源不同。但做到第三个的时候我停下来了——因为我意识到自己在**第三次写同一套东西**。

## 共同的敌人：拍脑袋

三个职能，每天都在做高风险判断：这个方向投不投、这个品牌到底强不强、这家公司值不值这个价。这些判断长期依赖同一个词——**感觉**。

「感觉」有两个致命的失败模式：

1. **单一视角偏见**。你只问一个人，哪怕是最资深的那个，他的盲区就是你的盲区。
2. **结论无法归因**。拍脑袋说「挺好的」，半年后崩了，你分不清是判断错了、证据不全、还是世界变了。

三个不同的职能，害的是同一种病。所以解药也应该是同一套。

## 同一套骨架

我在三个产品里反复搭的，是同一条流水线：

```
  调研    →    合议    →    打分    →    归因
  多路并行     多评委独立    多镜头可证伪   30/90/365
  取证据      互不相见      给出分数      时间轴回看
```

四步，逐个说为什么。

**多评委合议，而不是单个分析师。** 单个模型（或单个人）给结论，本质还是「单一视角」。所以每一步判断都不是一个 agent 说了算，而是一组各持立场的评委独立打分、必答反方——有人盯战略、有人盯证据、有人专门唱反调。合议之前评委互不相见，避免相互带节奏。这样出来的不是一个观点，而是一张**分歧地图**：哪条结论是共识、哪条撕得最凶，一目了然。

**可打分、可证伪，而不是「我觉得挺好」。**「这个品牌很强」是没法复盘的。「这个品牌在范畴命名维度 8.8 分，在真实信号维度只有 5.2 分，因为线上声量集中在单一渠道」——才是。每条结论都被要求给出分数和可证伪点：满足什么条件它就算错。判断从此可以被反驳，而不是只能被相信。

**版本化 + 归因校验，而不是一次性。** 判断打上时间戳，冻结成不可变快照，并预设 30 / 90 / 365 天的归因校验点。到期系统自己回来比对：当初的预判命中了没有？命中 / 证伪写进记分板。判断力于是第一次变成**可积累的资产**——你不只是做决定，你在给自己的判断建一条可回测的曲线。

**反捏造，是硬约束不是选项。** 大模型最危险的地方，是它能一本正经地编一个数字。所以三个产品都把「反捏造」做成机器级约束：只用公开一手 / 真实数据，拿不到就标 N/A 留白，绝不替你圆。OAF 干脆把数字全部交给确定性工具、只让模型写叙事；MBA 把评委引用接进 CI 硬 gate，逐字校验是否真在语料里。**宁可少说，不可乱说。**

**人机双读。** 同一份判断，既要能给人看（HTML 报告 / 看板），也要能给 agent 调（MCP / API）。因为在一个 AI 原生的组织里，判断的下游既有人，也有别的 agent。

## 换的，只是评委和数据源

看懂了骨架，三个产品的差异其实很表层：

- **Boss** ｜ 评委：维度 doctrine + 锚点心证 ｜ 数据源：战略议题、内部文档
- **MBA** ｜ 评委：人物 mental model（傅盛 / 乔布斯…）｜ 数据源：品牌信号、舆情
- **OAF** ｜ 评委：多空辩论 + 确定性工具 ｜ 数据源：三市场行情财报

骨架一行没变。这也是为什么它们能凑成**一套**，而不是三个孤立工具。

## 为什么是「AI 原生组织」

再往上想一层。

一个 AI 原生组织，是把越来越多的日常运转——写代码、跑调研、做客服、盯数据——交给 agent 的组织。当执行层已经跑在可追溯的 agent 上，**唯独顶层判断还停留在会议室的直觉里**，就成了最刺眼的断层。

C-suite 决策智能体想补的就是这一层：让 CEO / CMO / CFO 的核心判断，长在和组织其余部分同一套「可调研、可打分、可复盘」的底座上。不是取代人拍板——**最终按下按钮的还是人**——而是把拍板之前的调研、合议、打分、归因，变成一套人和 agent 都能读、都能回看的基础设施。

组织越 AI 原生，这层基础设施就越不可少。

## 把这套骨架压成一个图形

最后，给这套东西做了一枚标识——**三弧一点，一枚印**：

> 〔配图：C-suite 印章 logo — 用文末附的 c-suite-mark.png 上传〕

三个图形元素，各压着一条信条：

- **一枚印章。** 站点的朱砂色本来就是印泥的颜色，而「盖章」正是决策的终极动作。这套产品讲版本化冻结——每份判断书生成不可变快照——印章天然携带这层意思：盖下去，就是一个不可篡改的版本。
- **三段断开的弧，合成一个 C。** 三段弧是三个席位（CEO / CMO / CFO）；断开，是评委独立打分、互不相见；合起来读作一个 C，是合议成一体、一套骨架。开口朝右——开放的判断，不闭环自证。
- **中心一点墨。** 全图唯一不是朱砂的元素。它是锚点：最终按下按钮的那个人。机器围绕着人转，不是取代人拍板——正是上一节说的那件事，画进了图形里。

## 边界

说点诚实的。这套东西不是神谕：

- 评委是 AI 演绎的视角，不是真人董事会；它逼你想全，但不替你负责。
- 打分是相对的、给定语料下的，语料变了分数就该变——这正是要版本化的原因。
- 归因校验需要**时间**，30/90/365 天没到，记分板就还是空的。

但即便如此，把判断从「拍脑袋」挪到「可追溯的流水线」，本身就已经是一次升级。剩下的，交给时间去回测。
