## 是什么

Boss（bossagent.cc）是给领导者的一套**战略 OS**——把"锚点（某位领导者 / 决策者）的判断方法论"工程化的决策智能体。你发一个战略议题、或附一份文档，系统用 **1 个锚点心证 + N 个各持一套 doctrine 的维度评委**独立打分、必答反方，几分钟产出一份可引用、可打分、可证伪的判断报告。

- **多评委独立合议**：行业趋势 / 战略目标 / 客户 / 产品 / 组织 / 经营各一位评委，各持方法论、互不相见地打分
- **5 镜头打分**：推理 / 证据 / 反方 / 可证伪 / 韧性，每个判断必含一条可证伪的 metric
- **anchor_delta 机制**：评委加权分与锚点心证的差值当作"镜子"，偏差过大就报警
- **30 / 90 / 365 天归因校验**：判断到期自动比对预判与实际结果，命中 / 证伪写进记分板
- **版本化冻结**：每份判断书生成不可变快照（chmod 444），供投委 / 审计事后验真；证伪的判断进 Failure Card 归档

## 最近进展

- **v1.17 boss-as-a-Service 上线**：判断引擎从单机 CLI 升级成可被外部程序和访客直接调用的服务，MCP + HTTP **双协议同核**，从方案到公网可体验一路闭环。
- **零配置在线体验台**：访客免注册、免 API key，直接在主页发议题就能跑全链多评委合议，三层成本护栏（默认关 + 日配额 + 单并发）兜底。
- **引擎抽库重构**（`boss_core` 纯函数核）：错误处理、文档 IO、prompt 装配、打分聚合、知识库查询全下沉成独立包，CLI 与服务共核不共壳，字节级快照证明零漂移。
- **公开站重构**：导航从 11 项精简到 7 项，新增 MCP 接入指南、飞书机器人落地页、判例库（提交的议题与报告公开保留 7 天）。
- **工程护栏拉满**：全量脚本对抗式安全审计（60 项发现结清）、LLM 多端点冗余与自动 failover、出站脱敏双闸 fail-close；测试规模 1787 passed，约 5.6 万行代码。

## 为什么做

重要决策常常是"拍脑袋 → 事后无法复盘 → 判断力无法沉淀"。Boss 把一次判断拆成结构化的合议：不是问一个人，而是让一组各持立场的评委独立打分、必答反方；不是给个结论就完，而是给每条结论标好可证伪点和归因校验时间，到期回来验证当初判断是否成立。判断力于是从"感觉"变成可积累的资产。

数据默认保密（confidential by default）：真实案例、报告、内部知识库不出内网，公开层只放方法论。

## 你能怎么用

- **在线体验台**：主页直接发议题，零配置跑一遍多评委合议。
- **MCP / HTTP 服务**：用自己的模型不限量跑（服务化档位下 Boss 不调 LLM，由调用方模型跑合议）。
- **Claude Code Skill**：本地装 `/boss` skill + CLI 部署。
- **飞书机器人**：单聊或群里 @ 机器人即触发，PDF / Word / PPT / MD 多格式文档直接摄入。

适合：需要沉淀个人判断力的创始人 / 高管（当"锚点"）、主持决策的项目主理人、做事后审计的投委与外部审计。
